# 멀티 브라우저 격리 동작 수동 검증 가이드

본 문서는 Alert Watch의 다중 사용자(클라이언트) 환경 격리 기능이 정상적으로 작동하는지 로컬 브라우저에서 직접 검증하는 방법을 안내합니다.

현재 코드베이스는 `server.js`와 `scraper.js`가 각 클라이언트의 고유 `clientId`(`localStorage` 기반)를 기준으로 독립된 설정(`config.json`)과 감시 프로세스(`activeMonitors` Map)를 개별 관리하도록 구현되어 있습니다.

---

## 1. 수동 검증 준비 단계

1. **로컬 서버 구동**
   프로젝트 루트 디렉토리에서 터미널을 열고 아래 명령어로 서버를 기동합니다.
   ```bash
   npm run dev
   ```
   서버가 정상 구동되면 `http://localhost:3000` 주소로 접속이 가능해집니다.

2. **서로 다른 브라우저 세션 준비**
   세션 및 `localStorage` 격리 상태를 테스트하기 위해 아래와 같이 독립된 브라우저 창 2개를 실행합니다.
   - **브라우저 A (일반 창)**: 크롬 또는 사파리 일반 모드 창.
   - **브라우저 B (시크릿 창)**: 동일 브라우저의 시크릿 모드(Incognito) 창 또는 다른 종류의 브라우저(Edge, Firefox 등).

---

## 2. 격리 동작 검증 시나리오

두 브라우저에서 서로 다른 설정을 입력하고 시작했을 때, 상호 간섭 없이 격리되어 동작하는지 확인합니다.

### 1단계: 브라우저 A 설정 및 감시 시작
1. **브라우저 A**로 `http://localhost:3000`에 접속합니다.
2. 설정 영역에 아래 정보를 입력합니다.
   - **대상 URL**: `http://localhost:3000/mock-product` (로컬 테스트용 가상 상품 페이지)
   - **상태 키워드**: `품절`
   - **감지 조건**: `'품절'이 사라지면 알림`
   - **확인 주기**: `10초`
3. **[설정 저장 및 감시 시작]** 버튼을 클릭합니다.
4. **결과 확인**: 현황 탭에 감시 상태가 `감시 중`으로 표시되고, 가상 상품이 현재 품절 상태이므로 탐지견 배지가 `품절 (SOLD OUT)` 상태를 가리키는지 확인합니다.

### 2단계: 브라우저 B 설정 및 감시 시작 (독립 설정 검증)
1. **브라우저 B**로 `http://localhost:3000`에 접속합니다.
2. **브라우저 A의 설정이 브라우저 B에 나타나지 않고 초기 화면(설정 비어있음)이 유지되는지 확인합니다.** (데이터 격리 확인)
3. 브라우저 B 설정 영역에 아래와 같이 다른 정보를 입력합니다.
   - **대상 URL**: `http://localhost:3000/mock-product`
   - **상태 키워드**: `존재하지않는키워드` (Mock 페이지에 존재하지 않는 임의의 문구)
   - **감지 조건**: `'존재하지않는키워드'가 사라지면 알림`
   - **확인 주기**: `10초`
4. **[설정 저장 및 감시 시작]** 버튼을 클릭합니다.
5. **결과 확인**: 브라우저 B는 입력한 키워드가 웹페이지에 원래 존재하지 않으므로, 저장 및 시작 즉시 `구매 가능 (AVAILABLE)` 판정을 내리고 알림이 오는지 확인합니다.

### 3단계: 두 브라우저 상태 교차 확인 (실시간 격리 검증)
1. 브라우저 B가 `AVAILABLE` 상태로 변했음에도 불구하고, **브라우저 A는 여전히 `SOLD_OUT` 상태를 정상 유지하고 있는지 확인합니다.**
2. 이 시점에 프로젝트 루트 폴더의 `config.json` 파일을 열어봅니다.
   - 파일 내부에 `client_xxxx` 형태의 키가 2개 생성되어 있고, 각각 A와 B에 입력한 서로 다른 설정 값(`keyword`, `condition` 등)이 독립적으로 기록되어 있는지 확인합니다.

---

## 3. 검증 완료 기준

- [ ] 브라우저 B에서 감시 설정을 변경하거나 시작/정지하더라도 브라우저 A의 감시 주기와 결과 상태에 아무런 영향이 없습니다.
- [ ] `config.json` 설정 파일에 다중 클라이언트 ID별로 설정이 안전하게 분할 저장됩니다.
- [ ] 서버 재시작 시에도 두 브라우저의 모니터링 프로세스가 각각 기존 상태 그대로 자동 복구됩니다.
