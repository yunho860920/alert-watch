알림 수신 채널 설정

재입고 발생 시 실시간으로 알림을 전달받을 경로와 연동 방법입니다.

1. PWA (기기 푸시 알림)

확인하는 중

현재 기기 브라우저나 설치형 웹 앱(PWA)을 통해 바탕화면 팝업 알림을 수신합니다.

PWA 알림 수신 방법:
  • iOS (아이폰 16.4+): 반드시 Safari 브라우저로 접속 ➡️ 공유 버튼 ➡️ [홈 화면에 추가]로 설치한 뒤, 홈 화면 앱으로 실행하여 [알림 켜기]를 등록해야 팝업이 활성화됩니다. (인앱/크롬 제한 있음)
  • Android / PC: 크롬/사파리 브라우저에서 '이력 & 알림' 탭의 [알림 켜기]를 등록하면 즉시 수신 가능합니다.

2. 텔레그램 메시지 알림

확인하는 중

서버와 텔레그램 봇을 연동하여 기기 종류와 앱 환경 제한 없이 실시간 메시지로 전달받습니다.

텔레그램 봇 연동 방법:
  1. 텔레그램 검색창에 @BotFather를 검색하여 대화방을 엽니다.
  2. /newbot을 입력한 뒤 가이드에 따라 봇 이름과 Username을 지정하여 HTTP API Token을 발급받습니다.
  3. 생성한 내 봇과의 대화방에 들어가서 /start 및 아무 메시지나 보냅니다.
  4. @userinfobot 또는 ID 조회 봇을 검색하여 본인의 채팅방 ID (Chat ID)를 알아냅니다.

3. 이메일 알림

확인하는 중

SMTP 메일 서버와 연동하여 지정해 둔 수신용 메일 주소로 즉시 알림 메일을 발송합니다.

이메일 SMTP 연동 방법:
  1. 네이버/구글 등 이용하시는 메일의 환경설정에서 SMTP 사용 설정을 켜고 앱 비밀번호 (2단계 인증용)를 생성합니다.
  2. 발신 메일 주소로 연동할 포털 사이트 메일 주소를 입력하고, 해당 사이트에서 발급받은 16자리 앱 암호를 '앱 비밀번호'에 입력해야 연동에 실패하지 않습니다.

4. 앱인토스 알림 (Toss Push)

미확인

토스 미니앱의 알림 수신 동의를 활성화하여 토스 앱 푸시 및 알림톡으로 알림을 수신합니다.

고급 감시 필터
비워두면 상품 전체 상태 변화를 감시합니다.
설정은 저장됐지만 이 기기는 아직 알림을 받지 않습니다. 현황 탭에서 알림을 켜주세요.

로컬 테스트 시뮬레이터

불러오는 중

로컬 가상 상품의 상태와 옵션을 변경하며 감시 동작과 푸시 알림 수신을 즉시 테스트해 보세요.

가상 상품 옵션별 상태 설정
전체 상품 상태 제어
도움말 및 백그라운드 절전 방지 팁
  • 품절 감지 테스트 (가상 상품): 외부 사이트 품절 상태를 제어할 수 없는 경우, 로컬 가상 상품 페이지를 URL로 등록하여 테스트할 수 있습니다. [Mock 상품 페이지 열기]
  • 백그라운드 지속 감시: 모니터링 엔진은 브라우저를 닫아도 백엔드 서버에서 24시간 작동하며 상태 변화 시 채널(PWA, 텔레그램, 메일)로 즉시 전송합니다.
  • 알림 대상 등록: 현황 화면의 [알림 켜기] 또는 '이력 & 알림' 메뉴에서 현재 브라우저를 알림 대상으로 등록하면 실시간 PWA 푸시를 받을 수 있습니다.
  • Render 서버 절전 문제: 무료 Render 호스팅 사용 시 15분 접속이 없으면 서버가 절전 상태로 돌입합니다. 24시간 감시를 이어가려면 UptimeRobot 서비스에 가입하신 후, 이 서버의 헬스체크 주소(/api/health)를 5분 주기로 찌르도록 등록해 두세요.
